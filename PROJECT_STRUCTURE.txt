=======================

   Main Astro frontend that powers spellingbeesolver.dev.

2. worker-backend-cloudflare/
   Main Cloudflare Worker + D1 + KV backend.

3. functions/
   Cloudflare Pages Functions used for same-domain API helpers such as archive detail loading.

   - generate-site-data.mjs

   - purge-analysis-cache.ps1

5. .github/workflows/
   - daily-rebuild.yml: rebuild/deploy pipeline triggered manually or by repository_dispatch from the worker.

DEPLOYMENT PATH
===============

1. Worker deploy:
   worker-backend-cloudflare/wrangler.toml

2. Static site build:
   npm run build

3. Pages deploy target:
   Cloudflare Pages project "spellingbee-solver"


IMPORTANT NOTES
===============

1. The worker is the analytics source of truth. The frontend should consume worker data, not re-create history math locally.
   Update 2026-05-23: the public site now builds from generated snapshot files, so browsers should not call the worker directly for normal page rendering.

2. The site uses an ANALYTICS_CACHE KV namespace for per-puzzle cached analysis blobs.

3. The daily rebuild pipeline is intended to be:
   Cloudflare cron scrape -> worker stores puzzle -> worker triggers GitHub repository_dispatch -> GitHub Action backfills definitions -> GitHub Action generates public/site-data snapshots -> GitHub Action rebuilds + deploys Pages.

4. There is still some historical corpus drift versus nytbee on a handful of dates because the archive is broader than the exact official nytbee corpus.

5. Current public archive URL shape:
   /archive
   The selected puzzle detail loads inside that one page instead of using thousands of separate archive URLs.
